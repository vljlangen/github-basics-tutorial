Chapter 4 Authentication Setup

GitHub requires a Personal Access Token (PAT) instead of passwords for Git operations.

4.1 Preferred Method: R + RStudio

The easiest way to set up GitHub authentication:

# In R or RStudio:
install.packages(c("usethis", "gitcreds"))
usethis::create_github_token()  # Opens browser to create PAT
gitcreds::gitcreds_set()        # Enter your PAT when prompted

That’s it! The usethis::create_github_token() function will open your browser to create the PAT with the right permissions, and gitcreds::gitcreds_set() will store it securely.

4.2 Alternative: Git CLI + Credential Helper

If you prefer using Git from the command line:

macOS:

git config --global credential.helper osxkeychain

Windows:

git config --global credential.helper manager-core

When you clone or push, you’ll be asked for: - Username: your_github_username - Password: your_PAT_here

✅ Git will remember the token after this — no need to reenter.

4.3 Creating Your Personal Access Token Manually

If you need to create a PAT manually (instead of using usethis::create_github_token()):

Go to GitHub Settings → Developer settings → Personal access tokens → Tokens (classic), then:

  • Click “Generate new token (classic)”
  • Give it a descriptive name (e.g., “Academic Projects 2024”)
  • Set expiration (recommended: 90 days for learning, 1 year for regular use)
  • Select scopes:
    • repo - Full control of private repositories (most important)
    • workflow - Update GitHub Actions workflows
    • user - Update user data
  • Click “Generate token”
  • Copy the token immediately - you won’t see it again!

4.3.1 Token Security Best Practices

Do: - ✅ Use descriptive names for tokens - ✅ Set appropriate expiration dates
- ✅ Use minimum required scopes - ✅ Store tokens securely (password manager) - ✅ Revoke unused tokens

Don’t: - ❌ Share tokens with others - ❌ Commit tokens to repositories - ❌ Use tokens in public scripts - ❌ Give tokens more permissions than needed

4.4 Testing Your Setup

4.4.1 Verify Git Configuration

git config --list
# Should show your name and email

4.4.2 Test Authentication

Try cloning a repository:

git clone https://github.com/username/repository.git

When prompted, enter: - Username: your_github_username - Password: your_PAT (not your account password!)

If using the credential helpers above, Git will remember your token for future use.

4.5 Troubleshooting

4.5.1 Authentication Failed

Common issues: - Using account password instead of PAT - Token has expired - Insufficient token permissions - Typo in username or token

Solutions: 1. Verify you’re using the PAT, not your password 2. Check token expiration date 3. Ensure token has “repo” scope 4. Try clearing credentials and re-entering

4.5.2 Clear Stored Credentials

macOS:

git config --global --unset credential.helper

Windows: - Open Credential Manager - Remove GitHub entries - Or use: git config --global --unset credential.helper
