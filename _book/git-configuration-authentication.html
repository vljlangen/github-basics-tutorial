Chapter 2 Git Configuration & Authentication

2.1 Step 1: GitHub Account Setup

First, create your GitHub account (if you don‚Äôt have one):

  1. Go to github.com
  2. Sign up for a free account
  3. Choose a username (this will be part of your website URL)

2.2 Step 2: GitHub Authentication (Required)

GitHub requires a Personal Access Token (PAT) instead of passwords for Git operations.

Create your PAT: https://github.com/settings/tokens

üí° Note: If you choose the R approach below, usethis::create_github_token() will open this page for you automatically.

2.3 Step 3: macOS Authentication Setup

2.3.1 Option 1: R + RStudio + usethis + gitcreds

Best for R users:

# In R or RStudio:
install.packages(c("usethis", "gitcreds"))
usethis::create_github_token()  # Opens browser to create PAT
gitcreds::gitcreds_set()        # Enter your PAT when prompted

2.3.2 Option 2: Git CLI + macOS Keychain

Best for command-line Git users:

# Enable secure credential storage
git config --global credential.helper osxkeychain

üìå This ensures your PAT is stored in the macOS Keychain and reused silently.

When you clone or push, you‚Äôll be asked for:

  • Username: your_github_username
  • Password: your_PAT_here

‚úÖ Git will remember the token after this ‚Äî no need to reenter.

2.4 Step 4: Windows Authentication Setup

2.4.1 Option 1: R + RStudio + usethis + gitcreds

Best for R users ‚Äî identical process as on macOS:

# In R or RStudio:
install.packages(c("usethis", "gitcreds"))
usethis::create_github_token()  # Opens browser to create PAT
gitcreds::gitcreds_set()        # Enter your PAT when prompted

2.4.2 Option 2: Git CLI + Windows Credential Manager

Best for command-line Git users:

# Set the credential helper
git config --global credential.helper manager-core

üí° This tells Git to use Git Credential Manager (GCM), which stores credentials securely in Windows Credential Manager.

When you clone or push, you‚Äôll be asked for:

  • Username: your_github_username
  • Password: your_PAT_here

‚úÖ Git will remember the token after this ‚Äî no need to reenter.

2.5 Creating Your Personal Access Token

2.5.1 Step-by-Step PAT Creation

  1. Go to GitHub Settings ‚Üí Developer settings ‚Üí Personal access tokens ‚Üí Tokens (classic)
  2. Click ‚ÄúGenerate new token (classic)‚Äù
  3. Give it a descriptive name (e.g., ‚ÄúAcademic Projects 2024‚Äù)
  4. Set expiration (recommended: 90 days for learning, 1 year for regular use)
  5. Select scopes:
    • repo - Full control of private repositories (most important)
    • workflow - Update GitHub Actions workflows
    • user - Update user data
  6. Click ‚ÄúGenerate token‚Äù
  7. Copy the token immediately - you won‚Äôt see it again!

2.5.2 Token Security Best Practices

Do: - ‚úÖ Use descriptive names for tokens - ‚úÖ Set appropriate expiration dates
- ‚úÖ Use minimum required scopes - ‚úÖ Store tokens securely (password manager) - ‚úÖ Revoke unused tokens

Don‚Äôt: - ‚ùå Share tokens with others - ‚ùå Commit tokens to repositories - ‚ùå Use tokens in public scripts - ‚ùå Give tokens more permissions than needed

2.6 Testing Your Setup

2.6.1 Verify Git Configuration

git config --list
# Should show your name and email

2.6.2 Test Authentication

Try cloning a repository:

git clone https://github.com/username/repository.git

When prompted, enter: - Username: your_github_username - Password: your_PAT (not your account password!)

If using the credential helpers above, Git will remember your token for future use.

2.7 Troubleshooting

2.7.1 Authentication Failed

Common issues: - Using account password instead of PAT - Token has expired - Insufficient token permissions - Typo in username or token

Solutions: 1. Verify you‚Äôre using the PAT, not your password 2. Check token expiration date 3. Ensure token has ‚Äúrepo‚Äù scope 4. Try clearing credentials and re-entering

2.7.2 Clear Stored Credentials

macOS:

git config --global --unset credential.helper

Windows: - Open Credential Manager - Remove GitHub entries - Or use: git config --global --unset credential.helper
